Caesar Cipher

A simple and classic way to encode messages. Caesar Cipher is a substitution cipher, meaning that the original letter will be replaced by another letter or symbol. Caesar involves shifting letters based on a certain key (a user input) and the shifted letter will be the original letter's encryptment. The most typical key for Caesar is 3.

Method:
Let's say that the inputted text to decode was "Bill," and the shift key is 3. This means that A=D, B=E, C=F.
Encoded message:
Eloo

Morse Code

A famous Cipher known for its dots and dashes. This can be used in any form, whether it be the flashing of light, the beeping of doors, steps, winks, hands, etc. It is a simple substitution cipher than can use 1-6 characters for just one original character, and each character/letter is a different, unique combination of dots and dashes/underscores, with slashes acting as a space. This, however, does not include lowercase letters, and all messages are decoded into uppercase.

Method:
There is only one set of combinations for this cipher, meaning that it can be memorized, or referenced in a book or the internet. Let's say that we have the phrase, "I like triangles," and encode it into morse code.
Encoded message:
.. / .-.. .. -.- . / - .-. .. .- -. --. .-.. . ...

Keyword Cipher

Another simple substitution cipher, but this time involves the user thinking of a keyword. This keyword will then be placed in the front of the new substituted letters, and the letters that go after that will follow their basic alphabetical order. This does not have the use of lowercase letters. If there are duplicate letters in the word (i.e. BILLY), then the duplicate is removed (will be BILY).

Vigenere Cipher

A slightly trickier substitution cipher that also involves a keyword. This is shown through a square where each column and row has a letter represented to them, and the letter is the first to be seen in the column/row. It still follows the alphabetical order, and the message will be encrypted by the column, and the keyword by the row. The intersection of column and row will be the substituted letter for the original letter. This only uses uppercase letters.

A1Z26

A substitution cipher wherein each letter corresponds to a number according to their placement in the alphabet. For example: A=1, B=2, C=3. Applied, "Armageddon" would be "1-18-13-1-7-5-4-4-15-14" and "23-5-9-18-4" would be "Weird."

Rail Fence Cipher

A cipher that involves a table, and primarily rows. depending on the key (row) inputted by the user, its sentence can vary, and the message will be placed in the table in a zigzag-like formation. The message is then read out starting from row 1 until row n, in which that will be the ciphered message output.
For example, our message will be "Sixty Degrees," and our key is 3, so there will be three rows. Here is an image for its visualization:

Affine Cipher

A cipher wherein each letter first corresponds to a number from 0-25 (based on their order in the alphabet). Let this letter's corresponding number be x. Then, if your first key input number is a and your second key input number is b, a is multiplied to x and b is added to the product. Get this result and get the modulus of this divided by 26 (for the 26 letters of the English alphabet.) The resulting number/s is your encrypted message. Bacon Cipher

This cipher applies binary code into their function. But unlike normal binary, this cipher uses two letters to showcase the encrypted message rather than ones or zeroes. These two letters are commonly "a" and "b", and the binary string represents the index of the order of the alphabet (i.e. A is index 0, so the function shows "aaaaa"). Multitap Phone SMS

This is based on the buttons of multitap phones. In a 12 button keypad, 8 of them (numbers 2-9) can be pressed a different number of times to correspond to a letter. So, in written code, the number of times a certain number is pressed corresponds to a letter of the alphabet. For example, C is 222 because you have to press the number 2 button three times to get the letter C when typing on a multitap phone SMS.
